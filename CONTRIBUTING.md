# Contributing

Thanks for taking the time to contribute! This project prioritizes clarity and learning over completeness.

## Workflow

1. Check the [Issues](https://github.com) tab or the [ROADMAP.md](./ROADMAP.md) to see what has been done.
2. Create a branch from `main` following the [Conventional Commits](https://conventionalcommits.org) pattern.
3. Follow the "Running locally" section in the [README.md](./README.md) and keep the existing code style and conventions.

## Pull Requests

Before opening a PR:

- Make sure the project builds and runs locally.
- Review your own change.
- Add a clear description of what was done.
- Follow the Pull Request template (Overview and Features).

> [!NOTE]
> Direct pushes to `main` are restricted.

## Discussions

For major changes, feel free to open an issue first to discuss what you would like to change. So we can align on the approach properly.
