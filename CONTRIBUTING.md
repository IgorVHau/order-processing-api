# Contributing

Thanks for taking the time to contribute. 

## Getting started

- Check the [Issues](https://github.com) tab or the [ROADMAP.md](./ROADMAP.md) to see what has been done.
- Create a branch from `main` using a descriptive name according to the [Branching](#branching) section:
  - `feat/...`
  - `fix/...`
  - `refactor/...`
  - `test/...`

You can follow the "Running locally" section in the [README.md](./README.md) to get the project up and running.

## Guidelines

A few simple things that help keep the project consistent:

- Keep changes easy to review
- Follow the existing code style and conventions
- Write clear and descriptive commit messages (Conventional Commits)
- Update documentation when necessary

## Branching

This project follows the [Conventional Commits](https://conventionalcommits.org) specification:

Common types:

- `feat`: new feature
- `fix`: bug fix
- `refactor`: internal improvements
- `test`: tests
- `chore`: maintenance / setup changes
- `docs`: documentation updates

## Pull Requests

Before opening a PR:

- Make sure the project builds and runs locally
- Review your own change
- Add a clear description of what was done
- Follow the Pull Request template provided by the repository

## Pull Request format

This repository uses a standard Pull Request template with:

- Overview
- Features

Please follow this structure when opening a PR.

## Notes

- All changes go through Pull Requests (direct pushes to `main` are restricted)
- This project prioritizes clarity and learning over completeness

## Discussions

For major changes, feel free to open an issue first to discuss what you would like to change. So we can align on the approach properly.

Thanks again for contributing!
